# 開発記録ディレクトリ (`_docs/develop`)

## 📋 目的

このディレクトリは、**AIがプロジェクトのコンテキストを理解しやすくするため**の重要な設計決定や問題解決の記録を残すためのものです。

## 🎯 記録すべき内容

### ✅ 記録すべきもの

以下のような**重要な変更**のみを記録してください：

1. **アーキテクチャの変更**
   - システム設計の大きな変更
   - データベーススキーマの重要な変更
   - 新機能の追加（複数ファイルにまたがる変更）

2. **重要な問題解決**
   - 本番環境で発生した重大な問題とその解決
   - パフォーマンス問題の解決
   - セキュリティ関連の修正

3. **設計思想の記録**
   - なぜその設計を選んだか
   - 代替案とその比較
   - 将来の開発者への参考情報

### ❌ 記録不要なもの

以下のような**細かい修正**は記録不要です（Gitコミットログで十分）：

1. **小さなバグ修正**
   - 単一ファイルの小さな修正
   - タイポ修正
   - スタイル調整

2. **既存機能の微調整**
   - UIの細かい改善
   - ログメッセージの変更
   - コメントの追加

3. **設定ファイルの変更**
   - `.env`の変更
   - 設定値の調整

## 📝 記録の形式

### ファイル名

```
YYYY-MM-DD_HHMM_タイトル.md
```

例: `2026-01-12_1822_GitHub訪問数データ取得の処理対象期間判定問題の修正.md`

### 記録の構造

```markdown
# タイトル

**日付**: YYYY年MM月DD日 HH:MM
**タイトル**: 変更の概要
**担当者**: 名前またはAI Assistant
**プロジェクト**: プロジェクト名

## 📋 問題の概要
（または変更の概要）

## 🔍 原因の分析
（問題解決の場合）

## 🛠️ 修正内容
（または実装内容）

## 📊 修正結果
（または実装結果）

## 🎯 まとめ
```

## 🔄 運用ルール

### チーム開発の場合

1. **レビュー時に確認**
   - 大きな変更の場合は、`_docs/develop`への記録をレビューで確認
   - 記録が必要かどうかは、チームで判断

2. **定期的な整理**
   - 四半期ごとに、古い記録を整理
   - 重要なものは`_docs/document`に移動
   - 不要なものは削除

3. **コミットメッセージの活用**
   - 詳細な説明はコミットメッセージに記載
   - `_docs/develop`は補足的な情報のみ

### AI開発の場合

1. **自動記録の判断基準**
   - 複数ファイルにまたがる変更
   - 本番環境で発生した問題
   - 設計思想の変更

2. **コミットログとの連携**
   - コミットメッセージに詳細を記載
   - `_docs/develop`は重要な背景情報のみ

## 💡 判断基準

**記録すべきか迷った場合**：

1. **「この変更を3ヶ月後に見返したとき、なぜこの設計にしたか分かるか？」**
   - YES → 記録すべき
   - NO → コミットログで十分

2. **「新しいメンバーがこのコードを見たとき、背景が分かるか？」**
   - NO → 記録すべき
   - YES → コミットログで十分

3. **「この変更は複数のファイルにまたがっているか？」**
   - YES → 記録を検討
   - NO → コミットログで十分

## 📚 関連ドキュメント

- `_docs/document/`: プロジェクトの構成や設計思想（静的ドキュメント）
- Gitコミットログ: すべての変更履歴
- `README.md`: プロジェクトの概要

## 🗑️ 整理方針

### 定期的な整理（四半期ごと）

1. **重要な記録**: `_docs/document`に移動して永続化
2. **不要な記録**: 削除（Git履歴に残るため問題なし）
3. **重複情報**: 統合または削除

### 整理の判断基準

- **3ヶ月以上経過した記録**: 重要性を再評価
- **参照されていない記録**: 削除を検討
- **Gitコミットログで十分な記録**: 削除を検討

---

**重要**: このディレクトリは**補助的な情報**を残すためのものです。すべての変更履歴はGitコミットログに記録されています。過度に記録を残すと、かえってノイズになります。**「本当に必要な情報だけ」を記録することを心がけてください。**
